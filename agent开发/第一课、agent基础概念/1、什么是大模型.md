# 什么是大模型
## 一、大模型本质
1. 核心构成：**参数文件**（存储知识权重）+**运行代码**（加载参数做前向传播）
2. 底层逻辑：输入文本，输出下一个词的概率分布，是纯数学函数；Agent所有功能都是对该函数的工程化包装

## 二、大模型训练三阶段
| 阶段 | 核心动作 | 数据特点 | 目的 | 与Agent关联 |
|------|----------|----------|------|-------------|
| 预训练 | 互联网文本有损压缩 | TB级、质量参差 | 学世界知识 | 幻觉源于压缩丢细节；模型越大压缩损失越小 |
| 微调（SFT） | 换高质量问答对训练 | 10万条级、人工编写 | 学助手行为 | System Prompt与微调逻辑一致；RAG补充知识准确性 |
| RLHF | 人类对模型答案排序优化 | 模型生成+人类标注 | 学优质回答标准 | Agent可借鉴做自我改进（保留优质运行结果） |
### 关键规律：Scaling Laws
模型效果由**参数量+训练数据量**决定，算力/数据越多效果越好；底座模型会持续变强，部分Agent问题可通过模型升级解决

## 三、Agent开发核心
1. 本质公式：**Agent = 大模型 + 工具**
2. 角色分工：大模型（大脑：理解问题、拆步骤、选工具、整结果）；工具（执行：搜索/计算/调API/读写文件）
3. 核心模式：ReAct（先想再做）、Plan-and-Execute（先规划再执行）
4. 多模态价值：打破文字输入输出限制，拓展Agent能力（视觉/语音/绘图Agent）

## 四、大模型特性→Agent开发要求
1. 知识系统缺陷：存在**反转诅咒**（单向索引），是“不可解释工件”→**必须加验证环节**（SQL校验、数据核对、人工确认）
2. 未来进化方向：System 2慢思考（深度推理、自我纠错）+ 自我改进→Agent推理能力、自主优化能力会持续提升

## 五、关键类比：LLM=操作系统
| 操作系统概念 | LLM/Agent对应 | 开发要点 |
|--------------|---------------|----------|
| 内存/RAM | 上下文窗口 | 珍贵有限，做好记忆管理/信息压缩 |
| 外设/工具 | 计算器/搜索引擎/代码解释器 | 工具调用是核心，最小化权限 |
| 内核态/用户态 | System Prompt/User Prompt | 合理设计提示词，定义Agent行为 |
| 进程调度/IPC | 任务编排/多Agent通信 | 优化多任务、多Agent协作逻辑 |
| 闭源/开源系统 | GPT/Claude / Llama/Qwen | 按需选型，利用开源生态降低成本 |

## 六、Agent安全开发
### 三类核心攻击
1. 越狱攻击：通过角色扮演/编码绕过安全限制
2. 提示词注入：外部内容藏恶意指令，劫持Agent行为（最相关）
3. 数据投毒：训练/微调数据植触发词，篡改模型行为
### 四大防御建议
1. 隔离外部内容，不将其作为可信指令
2. 工具权限最小化（只读不写、单表查询）
3. 关键操作（金钱/删数据/外通信）加人工确认
4. 模型生成的执行代码（SQL/脚本）先校验再运行

## 七、底层开发指导原则
1. 接受幻觉不可消除，用RAG/工具调用缓解
2. 不高估模型推理能力，让其专注决策而非执行
3. 重视上下文窗口限制，优化token使用
4. 安全设计前置，在架构初期加入防御措施
5. 依据Scaling Laws，做好底座模型选型

# Agent概念与基础核心笔记
## 一、Agent核心定义
Agent（智能体）是能**自主行动完成任务**的系统，依据目标自主调用工具、访问数据库、执行步骤，区别于传统“输入-输出”问答，核心具备**多步骤规划、工具使用、状态更新、自主决策**能力，无需持续人工指导。
**核心价值**：自动化人类工作流程，将人从重复性、规则性任务中解放，聚焦高阶决策与创造，而非取代人类。

## 二、Agent典型应用案例（4类）
### 1. 发票处理（简单规则型任务）
**核心目标**：自动提取发票关键信息并录入数据库
**流程**：PDF转文本（API）→LLM解析文本（识别发票、提取开票方/金额/到期日等字段）→调用工具更新数据库→生成成功提示
### 2. 回复客户邮件（固定步骤型任务）
**核心目标**：自动化处理客户投诉/咨询邮件
**流程**：LLM提取关键信息（订单号/产品/问题）→调用工具查询订单数据库→LLM起草回复草稿→人工审核→审核通过后自动发送
### 3. 通用客户服务Agent（动态决策型任务）
**核心难点**：处理开放式问题，**步骤无法预先固定**，需根据条件自主判断后续行动
**典型场景**
- 库存查询：查询对应商品库存→综合结果回复
- 退货处理：验证购买记录→检查退货政策→允许则生成退货标签+更新数据库状态
### 4. Manus/OpenClaw（复杂交互型任务）
**核心能力**：像人类一样使用浏览器完成复杂操作，需融合多能力
**典型任务**：查询航班空座（打开浏览器→填写表单/导航页面→搜索航班→确认结果）
**必备能力**：视觉理解、推理决策（遇错调整策略）、工具调用（模拟键鼠）、状态跟踪（记住进度）

## 三、Agent构建核心技能：任务分解
### 1. 核心四步方法论
1. **观察人类行为**：思考人类完成该任务的操作逻辑
2. **拆解子步骤**：将整体任务拆分为独立、清晰的子步骤
3. **评估可行性**：判断每个子步骤能否用LLM或工具实现
4. **迭代优化**：效果不佳则进一步细化子步骤，直至可落地
### 2. 经典案例拆解
#### 案例1：撰写深度论文（迭代细化思路）
- 直接生成：LLM一次性输出→问题：内容表面化、无深度
- 三步工作流：生成大纲→网络搜索→撰写论文→优化：补充信息，提升深度
- 五步工作流：生成大纲→网络搜索→撰写初稿→分析修订点→修订草稿→核心：模拟人类“写作-反思-修改”循环，大幅提升输出质量
#### 案例2：回复客户投诉邮件
- 拆解步骤：提取关键信息（LLM）→查询客户记录（工具）→撰写并发送回复（LLM+工具）
- 关键洞察：单一任务拆分为独立可执行的子步骤，由LLM和工具协同完成，形成自动化流程

## 四、Agent工作流必备两大基本组件
### 1. 模型（Models）：Agent的“大脑”
- **大型语言模型（LLM）**：核心主力，负责文本生成、工具使用决策、信息提取
- **其他AI模型**：处理非文本模态数据，如PDF转文本、文本转语音、图像分析等
### 2. 工具（Tools）：Agent的“手脚”
- **代码/命令工具**：编写运行代码、执行Bash命令、脚本访问外部接口
- **计算机操作工具**：网页搜索、获取实时数据（天气）、发送邮件、查看日历、模拟键鼠操作等
- **数据/业务工具**：数据库查询/更新、订单系统调用、库存查询、退货政策校验等

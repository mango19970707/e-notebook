# Agent设计模式核心笔记
## 一、Agent四大核心设计模式
### 1. 反思（Reflection）
**核心**：让模型对自身输出做**检查、评估、迭代改进**，是性价比极高的性能优化手段
**工作流程**：初始生成结果→自我/外部评估（分析问题、提改进建议）→基于反馈优化→循环至达标
**关键要点**
- 可单模型完成，也可引入独立“审查者”模型（多智能体协作雏形）
- 评估标准可客观（代码能否运行）、可主观（文案风格）
- 无法保证100%完美，但能带来**显著且稳定的性能提升**

### 2. 工具使用（Tool Use）
**核心**：赋予LLM调用外部工具的能力，打破纯文本生成限制，实现与现实世界的互动
**工作流程**：任务处理中识别工具需求→生成工具调用指令/参数→工具执行并返回结果→模型整合结果完成任务
**主流工具分类**
- 信息收集：网页搜索、维基百科、数据库访问
- 分析计算：代码执行、Wolfram Alpha、代码解释器
- 生产力：邮件、日历、即时通讯
- 图像处理：图像生成、图像描述、OCR识别

### 3. 规划（Planning）
**核心**：让模型**自主拆解复杂任务、决定工具调用顺序**，替代开发者硬编码流程
**工作流程**：理解复杂任务目标→自动规划步骤/工具调用序列→按规划依次执行
**关键要点**
- 比硬编码流程更灵活，支持模型自主构思新任务解决方案
- 控制难度更高、偏实验性，可能产出超出预期的结果
- 典型案例：根据示例图姿势生成指定图片并语音描述（拆分为姿势提取→图像生成→文本描述→语音转换四步）

### 4. 多智能体协作（Multi-agent collaboration）
**核心**：让**不同专长的智能体分配专属角色**，分工协作完成复杂任务
**工作流程**：角色分配（研究员/编辑/计算师等）→各智能体执行子任务→智能体间沟通协调、传递信息
**核心优势&缺点**
- 优势：复杂任务表现远优于单智能体（人物传记66.0%→73.8%、MMLU63.9%→71.1%、国际象棋29.3%→45.2%）
- 缺点：控制和调试难度大，无法提前预知各智能体行为
  **核心价值**：发挥不同模型的专长，实现“1+1>2”的协作效果

## 二、反思模式：落地方法与核心价值
### 1. 核心类比&设计逻辑
模仿人类工作的**“初稿-反思-修改”循环**（如写作、编程），并非魔法而是工程实践，是Agent开发中**实现简单、收益极高**的优化方法。

### 2. 典型落地案例
#### 案例1：写邮件（单模型硬编码反思流程）
- 初稿问题：表述模糊（next month无具体日期）、拼写错误、缺少署名
- 反思流程：将初稿输入模型+提示“反思并改进邮件草稿”→模型生成修正版（明确日期、改错别字、加署名）
- 关键：工程师预先设计步骤，属于**可靠、易落地**的工程化方法

#### 案例2：代码编写（多模型协作+外部反馈，终极形态）
1. 基础版：单模型生成代码→同模型检查并修复bug
2. 进阶版：快速生成模型写初稿→逻辑推理模型做审查改进（发挥各模型专长）
3. 终极版：生成代码→沙盒环境执行→捕获错误/输出结果（外部反馈）→模型结合反馈重写代码→产出可运行终稿

### 3. 反思模式的核心结论
1. 普适性强：在**所有任务、所有主流模型（GPT-3.5/4、ChatGPT、Claude）** 中，反思表现均碾压直接生成
2. 外部反馈是关键：引入工具执行结果、数据库查询、API返回等外部信息，能让反思更深入，突破纯自我推理的瓶颈
3. 设计哲学：只要有机会获取额外信息，就必须融入反思流程，是提升系统鲁棒性的核心策略

### 4. 高效反思提示语编写「两大黄金法则」
1. **明确指示反思动作**：不用模糊的“请改进”，改用“审查/检查/验证”+具体任务（如“审核电子邮件初稿”）
2. **具体指定检查标准**：不用笼统的“让它更好”，列出明确评判维度（如邮件：语气专业、事实准确；域名：易发音、无负面含义）

## 三、反思模式实战：图表生成工作流（多模态+多模型协作）
以**2024/2025年Q1咖啡销量对比图表生成**为例，实现从“粗糙初稿”到“专业可视化”的迭代，核心融合**反思模式+多模态模型+多模型分工**
### 1. 直接生成的问题
LLM生成绘图代码→运行产出堆叠柱状图，存在**对比不直观、观感不专业**两大缺陷，无法满足实际需求

### 2. 反思优化全流程
1. 输入准备：将初稿代码+生成的图表打包，作为多模态LLM的输入
2. 角色设定：提示多模态模型扮演**专业数据分析师**，进行批判性评估
3. 视觉推理：模型“可视化分析”图表问题，提出具体改进建议
4. 代码更新：模型根据反馈生成优化版绘图代码
5. 最终产出：运行优化代码，生成直观、专业的对比图表

### 3. 多模型分工策略
- 初始生成模型：通用大模型（如GPT-4），负责快速产出基础绘图代码
- 反思审查模型：强推理能力的“思考模型”，负责评估图表、提供改进方案并更新代码

### 4. 实战核心结论
反思机制效果因场景而异，需针对性测试；可通过**调整初始生成提示、优化反思提示**，最大化反思在特定场景的收益。

## 四、外部反馈：突破Agent性能瓶颈的关键
### 1. 性能提升的三层曲线（提示词工程收益递减规律）
以**提示词工程投入时间**为横轴，**系统性能**为纵轴，分为三个阶段：
1. **无反思**（红色）：初期调提示词性能快速提升，随后进入平台期，再投入时间无显著收益
2. **有反思**（蓝色）：加入反思后，性能再次上扬，突破原瓶颈，达到更高平台
3. **反思+外部反馈**（黄色）：接入外部反馈后，性能**二次跃升**，达到远超前两者的高度，实现质的突破

### 2. 外部反馈的核心价值
为系统注入**模型知识库外的新信息**，让Agent不再局限于自身推理，而是结合**工具执行结果、真实数据、外部接口返回**等客观信息做决策和优化，从根本上打破性能天花板。

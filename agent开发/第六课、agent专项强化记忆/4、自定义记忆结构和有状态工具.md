# 第四课：自定义记忆结构和有状态工具——让 Agent 按你的方式管理记忆 内容总结
本课核心为基于Letta框架实现Agent的**自定义记忆结构**与**有状态工具开发**，打破默认human/persona记忆结构限制，适配任务队列、项目管理等个性化业务场景，从“使用框架”进阶到“改造框架”。

## 一、记忆区段（Memory Block）核心属性
是Letta中记忆的基本单元，每个区段含4个唯一关键属性，同一Agent内label不可重复，id全局唯一支持跨Agent共享，limit限制字符长度防止占满上下文窗口。
| 属性 | 含义 |
| --- | --- |
| label | 区段标签，如"human"、"persona"，同一Agent内不能重复 |
| value | 区段实际内容，存入上下文窗口的文字 |
| limit | 字符上限，超过后core_memory_append调用失败 |
| id | 全局唯一ID，支持跨Agent共享记忆区段 |

**核心操作代码**：
```python
# 创建Agent时定义记忆区段
agent_state = client.agents.create(
    memory_blocks=[
        {
            "label": "human",
            "value": "用户名字:南哥"
        },
        {
            "label": "persona",
            "value": "你是一个友好的 AI 助手"
        }
    ],
    model="openai/gpt-4o-mini",
    embedding="openai/text-embedding-3-small"
)

# 查看所有记忆区段信息
blocks = client.agents.blocks.list(agent_id=agent_state.id)
for block in blocks:
    print(f"标签: {block.label}")
    print(f"内容: {block.value}")
    print(f"字符上限: {block.limit}")
    print(f"ID: {block.id}")
    print()

# 通过block ID访问区段
block_id = blocks[0].id
block = client.blocks.retrieve(block_id)
print(block.value)

# 通过Agent ID + label访问区段
human_block = client.agents.blocks.retrieve(
    agent_id=agent_state.id,
    block_label="human",
)
print(human_block.value)
```

## 二、记忆区段的上下文渲染（block prompt template）
Letta通过**block prompt template**控制记忆区段在上下文的渲染格式，默认为XML标签形式，会显示当前字符使用量/上限，Agent可据此判断是否需归档记忆；模板支持自定义修改（如Markdown格式），默认格式足够日常使用。

**核心操作代码**：
```python
# 查看当前渲染模板
template = client.agents.core_memory.retrieve(
    agent_id=agent_state.id
).prompt_template
print(template)

# 默认模板渲染后的XML格式示例
<human characters="12/5000">
用户名字:南哥
</human>
<persona characters="16/5000">
你是一个友好的 AI 助手
</persona>
```

## 三、有状态工具（Stateful Tools）
区别于无状态工具（仅接收参数、返回结果，不接触Agent内部状态），有状态工具可**访问并修改Agent的内部状态**，是自定义记忆管理的基础。核心特性为参数中声明`agent_state: "AgentState"`（系统自动注入，LLM无需感知，引号避免导入错误），工具可通过该参数获取Agent状态。

### 核心示例：获取Agent ID的有状态工具
```python
# 定义有状态工具
def get_agent_id(agent_state: "AgentState"):
    """
    Query your agent ID field
    """
    return agent_state.id

# 注册工具
get_id_tool = client.tools.upsert_from_function(func=get_agent_id)

# 创建使用该工具的Agent
agent_state = client.agents.create(
    memory_blocks=[], # 可无核心记忆区段
    model="openai/gpt-4o-mini",
    embedding="openai/text-embedding-3-small",
    tool_ids=[get_id_tool.id]
)

# 流式响应测试工具（实时返回每一步操作）
response_stream = client.agents.messages.create_stream(
    agent_id=agent_state.id,
    messages=[
        {
            "role": "user",
            "content": "你的 Agent ID 是什么?"
        }
    ]
)
for chunk in response_stream:
    print_message(chunk)
```
**关键特性**：流式响应（create_stream）对比一次性响应（create），支持实时查看Agent操作步骤；有状态工具的自动注入参数无需LLM生成，工具调用参数为空字典`{}`。

## 四、实战：自定义工具实现任务队列记忆
替换默认记忆结构，创建**tasks**记忆区段，开发`task_queue_push`（添加任务）和`task_queue_pop`（取出任务）自定义有状态工具，关闭默认工具，实现Agent的任务队列管理。

### 4.1 定义添加任务工具
```python
def task_queue_push(agent_state: "AgentState", task_description: str):
    """
    Push to a task queue stored in core memory.

    Args:
        task_description (str): A description of the next task you must accomplish.

    Returns:
        Optional[str]: None is always returned as this function does not produce a response.
    """
    from letta_client import Letta
    import json

    client = Letta(base_url="http://localhost:8283")

    # 获取tasks记忆区段
    block = client.agents.blocks.retrieve(
        agent_id=agent_state.id,
        block_label="tasks",
    )
    # 解析并追加新任务
    tasks = json.loads(block.value)
    tasks.append(task_description)

    # 写回更新后的任务列表
    client.agents.blocks.modify(
        agent_id=agent_state.id,
        value=json.dumps(tasks),
        block_label="tasks"
    )
    return None
```

### 4.2 定义取出任务工具
```python
def task_queue_pop(agent_state: "AgentState"):
    """
    Get the next task from the task queue.

    Returns:
        Optional[str]: Remaining tasks in the queue.
    """
    from letta_client import Letta
    import json

    client = Letta(base_url="http://localhost:8283")

    # 获取tasks记忆区段
    block = client.agents.blocks.retrieve(
        agent_id=agent_state.id,
        block_label="tasks",
    )

    tasks = json.loads(block.value)

    # 队列为空返回None
    if len(tasks) == 0:
        return None

    # 取出第一个任务
    task = tasks[0]

    # 更新剩余任务并写回
    remaining_tasks = json.dumps(tasks[1:])
    client.agents.blocks.modify(
        agent_id=agent_state.id,
        value=remaining_tasks,
        block_label="tasks"
    )
    return f"Remaining tasks {remaining_tasks}"
```

### 4.3 注册工具+创建任务队列Agent
```python
# 注册自定义工具
task_queue_push_tool = client.tools.upsert_from_function(
    func=task_queue_push
)
task_queue_pop_tool = client.tools.upsert_from_function(
    func=task_queue_pop
)

# 创建Task Agent
import json
task_agent = client.agents.create(
    system=open("task_queue_system_prompt.txt", "r").read(), # 自定义系统提示词
    memory_blocks=[
        {
            "label": "tasks",
            "value": json.dumps([]) # 初始化空任务队列
        }
    ],
    model="openai/gpt-4o-mini-2024-07-18",
    embedding="openai/text-embedding-3-small",
    tool_ids=[
        task_queue_push_tool.id,
        task_queue_pop_tool.id
    ],
    include_base_tools=False, # 关闭Letta默认工具
    tools=["send_message"] # 保留发消息工具
)

# 验证Agent工具列表和记忆状态
print([tool.name for tool in task_agent.tools])
print(client.agents.blocks.retrieve(
    task_agent.id,
    block_label="tasks"
).value)
```

### 4.4 运行测试任务队列
```python
# 给Agent派任务
response_stream = client.agents.messages.create_stream(
    agent_id=task_agent.id,
    messages=[
        {
            "role": "user",
            "content": "给你两个任务,分开添加:"
            "第一个是'以后叫我老王',"
            "第二个是'用我的名字写一首打油诗'"
        }
    ]
)
for chunk in response_stream:
    print_message(chunk)

# 验证任务队列是否清空
print(client.agents.blocks.retrieve(
    block_label="tasks",
    agent_id=task_agent.id
).value)

# 若任务未完成，催办检查
response_stream = client.agents.messages.create_stream(
    agent_id=task_agent.id,
    messages=[
        {
            "role": "user",
            "content": "还有没完成的任务吗?检查一下"
        }
    ]
)
for chunk in response_stream:
    print_message(chunk)
```
**执行逻辑**：Agent自主完成「任务入队→逐个取出执行→结果返回→队列清空」，共5个步骤，行为由LLM根据系统提示词自主决策。

## 五、自定义模式的延伸场景
基于**自定义记忆结构+有状态工具**的核心模式，可适配多类业务场景，核心思路为「自定义数据结构+自定义操作工具+Agent自主决策使用时机」：
1. 项目看板记忆：设todo、inprogress、done三个区段，开发工具移动任务卡片；
2. 客户状态追踪：用JSON结构的记忆区段记录客户信息，对话后自动更新字段；
3. 多Agent共享知识库：利用block的全局唯一ID，让多个Agent共享同一knowledge区段。

### 装饰器模式

---

- 1、应用场景

在不改变原有对象的基础上，动态地给一个对象添加一些额外的职责。

- 2、实现

 普通的妖怪上场之后喊sss，boss会先进行自我介绍，然后再sss
```go
package main

import "fmt"

type IAction interface {
	say()
}

type commonRole struct{}

func (cr commonRole) say() {
	fmt.Println("sss")
}

type boss struct{}

func (b boss) say(cr commonRole) {
	fmt.Println("i am boss xxx")
	cr.say()
}

func main() {
	cr := commonRole{}
	b := boss{}

	cr.say()
	b.say(cr)
}
```
